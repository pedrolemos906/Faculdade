
# Metodologia


A equipe adotou uma metodologia baseada em práticas ágeis e design centrado no usuário, combinando organização eficiente e desenvolvimento iterativo. O projeto foi desenvolvido no Visual Studio Code, com versionamento utilizando Git e hospedagem no GitHub. 
A organização da equipe foi realizada com o Miro. O processo de desenvolvimento ocorreu em ciclos curtos, envolvendo levantamento de requisitos, planejamento, implementação, testes e refinamento contínuo. A comunicação constante entre os membros permitiu alinhamento e resolução rápida de problemas, assegurando um fluxo de trabalho colaborativo e focado na entrega de uma solução eficiente e alinhada às necessidades dos usuários.


## Controle de versão



A ferramenta de controle de versão adotada no projeto foi o [Git](https://git-scm.com/), sendo que o [GitHub](https://github.com/GustavoBraga3/2026-1-p1-tiaw-g3-redes-sociais.git) foi utilizado para hospedagem do repositório.

O projeto segue a seguinte convenção para o nome de branches:

- `main`: versão estável já testada do software
- `unstable`: versão já testada do software, porém instável
- `testing`: versão em testes do software
- `dev`: versão de desenvolvimento do software

Quanto à gerência de issues, o projeto adota a seguinte convenção para etiquetas:

- `documentation`: melhorias ou acréscimos à documentação
- `bug`: uma funcionalidade encontra-se com problemas
- `enhancement`: uma funcionalidade precisa ser melhorada
- `feature`: uma nova funcionalidade precisa ser introduzida

Discuta como a configuração do projeto foi feita na ferramenta de versionamento escolhida. Exponha como a gestão de tags, merges, commits e branches é realizada. Discuta também como a gestão de issues foi feita.

### Processo

A equipe utilizou Design Thinking para compreender e resolver o problema, passando pelas etapas de empatia, definição, ideação, prototipagem e testes, garantindo que a solução fosse centrada nas necessidades reais dos usuários. O desenvolvimento foi realizado no Visual Studio Code, com versionamento via Git e GitHub.

Além disso, foi aplicado o framework Scrum, com organização em sprints e divisão de responsabilidades, permitindo entregas contínuas e melhorias constantes. As tarefas foram gerenciadas pelo Trello, garantindo organização, colaboração e acompanhamento do progresso do projeto.
